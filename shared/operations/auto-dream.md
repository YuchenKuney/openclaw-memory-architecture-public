# AutoDream - 记忆整合系统

_借鉴 Claude Code KAIROS 的 autoDream 机制_
_Created: 2026-04-15_

---

## 📖 概念

Claude Code 的 `autoDream` 是在用户空闲时（类似睡眠）自动运行的记忆整理进程：
- 合并观察结果
- 去除矛盾
- 将模糊洞察转化为确定事实
- 更新 MEMORY.md 索引

---

## 🎯 目标

让 OpenClaw 定期自动整理记忆：
1. 清理过期任务记录
2. 合并矛盾信息
3. 提炼关键知识
4. 更新索引

---

## 🔄 整合流程

### Phase 1: Orient（定向）
```
1. 读取 MEMORY.md（核心记忆）
2. 扫描 memory/index.md（索引）
3. 识别当前活跃主题
```

### Phase 2: Gather（收集）
```
1. 检查最近7天的日志
2. 识别需要整合的信息
3. 标记矛盾或过时的条目
```

### Phase 3: Consolidate（整合）
```
1. 合并同一事件的重复记录
2. 解决矛盾（保留最新+最可靠）
3. 将模糊洞察转化为确定事实
4. 更新相关主题文件
```

### Phase 4: Prune（修剪）
```
1. MEMORY.md 限制 ≤200行 / ≤25KB
2. 移动旧日志到 archive/
3. 删除过期临时文件
```

---

## 📁 归档规则

| 类型 | 保留位置 | 期限 |
|------|----------|------|
| MEMORY.md | 根目录 | 永久（需精简） |
| memory/index.md | memory/ | 永久 |
| 每日日志 | memory/YYYY-MM-DD.md | 30天 |
| 归档日志 | memory/archive/YYYY-MM/ | 永久 |
| 临时文件 | memory/.xxx | 删除 |

---

## 🚫 不整合的内容

| 类型 | 原因 |
|------|------|
| 失败的任务记录 | 不污染记忆 |
| 推测性信息 | 未经验证 |
| 敏感凭据 | 安全考虑 |
| 临时调试信息 | 无持久价值 |
