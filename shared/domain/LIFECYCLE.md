# Memory Lifecycle Manager
> 记忆生命周期管理系统

## 问题
- 只增不减 → 记忆膨胀
- 无更新机制 → 同一事实多个版本
- 无去重 → 重复内容浪费空间

## 解决方案

### 1. 时间策略（TTL）
| 类型 | 活跃期 | 归档期 | 最终删除 |
|------|--------|--------|----------|
| 日志 | 7天 | 7-90天 | 90天后 |
| 索引 | 实时更新 | - | 永久保留 |
| 技能 | 无过期 | - | 手动删除 |

### 2. 更新机制
```
新条目 → 检查是否存在同主题 → 是：更新 | 否：添加
```

### 3. 去重机制
```
新条目 → 计算内容哈希 → 检查是否重复 → 是：跳过或合并 | 否：添加
```

## 文件结构
```
memory/
├── index.md           # 主索引（实时更新）
├── 2026-04-16.md      # 当日日志
├── archive/           # 归档目录
│   └── 2026-03/       # 按月归档
└── .lifecycle.json    # 生命周期状态
```

## 使用方法
```bash
# 运行生命周期管理
python3 memory_lifecycle.py --check

# 清理过期日志
python3 memory_lifecycle.py --cleanup

# 查看统计
python3 memory_lifecycle.py --stats
```

## 触发时机
1. **每日心跳时** — 检查 + 清理
2. **添加日志前** — 去重 + 更新检查
3. **启动时** — 快速检查
