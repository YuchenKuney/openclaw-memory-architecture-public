# Web4.0 子系统详解

## 定位
受控的网页信息获取系统，包含爬虫、铁律、浏览器、沙箱容器

## 核心模块

| 模块 | 文件 | 职责 |
|------|------|------|
| Cookie注入器 | `web4_cookie_injector.py` | 域名铁律过滤+Google Cookie二次过滤 |
| 浏览器 | `web4_browser.py` | Playwright隐身浏览器 |
| 沙箱容器 | `web4_container.py` | Linux Namespace网络隔离 |
| 控制器 | `web4_controller.py` | AI调用API（research函数） |
| 烹饪引擎 | `web4_cooker.py` | cooking参数注入 |

## Cookie铁律过滤

### 三层过滤机制

**第一层：域名白名单**
```python
ALLOWED_DOMAINS = {
    "google.com", ".google.com",  # 搜索引擎
    "youtube.com", ".youtube.com",
    "shopee.com", ".shopee.com",  # 电商
    "tiktok.com", ".tiktok.com",
    "bloomberg.com", "forbes.com",  # 新闻
    ...
}
```

**第二层：敏感域名黑名单**
```python
BLOCKED_DOMAINS = {
    "paypal.com", "stripe.com",  # 金融
    "facebook.com", "instagram.com",  # 社交
    "mail.google.com", "inbox.google.com",  # 邮件
    ".gov", ".health",  # 政府医疗
    "slack.com", "notion.so",  # 工作平台
    ...
}
```

**第三层：Google Cookie二次过滤**
```python
# ✅ 允许（公开搜索Cookie）
"SNID", "SEARCH_SAMESITE", "OTZ", "NID"

# ❌ 阻止（登录认证Token）
"HSID", "SSID", "APISID", "SAPISID",
"__Secure-1PAPISID", "__Secure-3PAPISID"
```

## IronRuler 4条绝对铁律

| 铁律 | 内容 |
|------|------|
| 铁律一 | Google账号只用于模拟普通用户，禁止账号操作 |
| 铁律二 | 只允许特定公开域名，禁止登录态个人数据 |
| 铁律三 | 单域名间隔≥3秒，会话≤50次，随机滚动 |
| 铁律四 | 会话结束强制清空所有Cookie/缓存 |

## 沙箱容器隔离

```python
# Linux Namespace 隔离
- network namespace: 独立网络栈
- PID namespace: 独立进程空间
- mount namespace: 独立文件系统视图
```

## 使用示例

```python
    sites=["shopee.com"],

        "max_pages": 5,
        "strategy": "deep"

)
```

## 与Clawkeeper的关系

- Web4：负责网络层安全（URL访问、Cookie注入）
- Clawkeeper：负责文件层安全（workspace文件变化）
- 两者互补，共同构成AI安全体系
