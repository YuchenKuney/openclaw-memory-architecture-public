# Errors & Solutions - 错误解决方案

> 记录常见错误和解决方案，防止重复犯错

---

## 常见问题

### 邮件汇报错误 ⚠️
**问题**：只读标题就汇报，导致误判
**解决**：必须读取正文 → 核实 → 再汇报
**教训**：乱汇报会导致严重后果

### 服务器操作错误
**问题**：操作间隔太短被检测
**解决**：间隔2-3秒执行命令
**教训**：避免自动化触发反爬虫机制

### 记忆丢失
**问题**：记忆被意外格式化
**解决**：定期备份到GitHub仓库
**教训**：配置定时备份脚本

---

## 安全规则

1. **不安装来路不明的代码**
2. **不直接克隆外部仓库到生产环境**
3. **先在测试环境验证**
4. **汇报前多次检测确认**

---

## 安全最佳实践

1. **敏感信息不写入代码仓库**
   - Token、Webhook、API Key 使用环境变量
   - 密码使用密钥管理服务
   
2. **日志不记录敏感内容**
   - 不记录密码、token、cookie
   - 错误信息脱敏处理

3. **定期轮换凭证**
   - Webhook 定期更换
   - API Key 定期更新
