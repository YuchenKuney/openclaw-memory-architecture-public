# Architecture - 系统架构设计

> 本架构借鉴 Claude Code + MemGPT + GraphRAG + Generative Agents

---

## 🎯 设计目标

1. **容量自适应** - 不受限于 LLM 上下文窗口
2. **层次分明** - 不同类型信息不同访问模式
3. **自动进化** - 从日志中学习，提炼为规则和信念
4. **可解释** - 每个决策都有记忆依据

## 🔧 核心模块

### 1. Orchestrator（调度器）
- 分析用户输入，确定需要调用哪些模块
- 控制记忆流向：记录 → 提炼 → 归档
- 协调 Rule Manager、Context Builder、Watchdog 的执行顺序

### 2. Context Builder（上下文构建器）
- 根据当前任务从三层记忆中动态构建上下文
- 优先级：L1 > L2 匹配 > L3 搜索
- 使用语义相似度排序

### 3. Rule Manager（规则管理器）
- 从日志中提取规则，计算置信度
- 支持偏好学习、行为规范、错误教训
- 置信度低于阈值时自动降级

### 4. Memory Watchdog（记忆看门狗）
- 定时检查 MEMORY.md 大小，超限预警
- 定时归档旧日志
- 定时执行日志提炼

### 5. Knowledge Graph（知识图谱）
- 实体 + 关系建模
- 支持 GraphRAG 风格的全局/局部查询
- Leiden 社区检测（可选增强）

## 📊 数据流

### 写入流程
```
用户交互
    ↓
日志写入 (L3: memory/YYYY-MM-DD.md)
    ↓
每日提炼 → Topic Files (L2)
    ↓
定期归纳 → MEMORY.md (L1)
    ↓
反思提炼 → Belief System
```

### 读取流程
```
用户查询
    ↓
Orchestrator 分析意图
    ↓
┌─ 精确问题 → KG Local Search
├─ 概括问题 → KG Global Search  
└─ 简单查询 → L1 直接返回
    ↓
Context Builder 构建上下文
    ↓
LLM 推理 + 回复
```
