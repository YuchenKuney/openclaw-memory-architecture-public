## 比赛演示路径

### 场景1：用户发送市场调研任务
```
用户消息（飞书）
    ↓
Orchestrator 接收
    ↓
Context Builder 构建上下文（三层记忆）
    ↓
Rule Manager 检查铁律
    ↓
Web4 Controller.research() 执行爬虫
    ↓
Clawkeeper 监控文件变化
    ↓
结果写入 results/
    ↓
飞书卡片推送进度给坤哥
```

### 场景2：AI尝试读取敏感文件
```
AI 读取 ~/.gitcredentials
    ↓
Clawkeeper watcher 检测到 READ 事件
    ↓
detector 正则规则命中 HIGH
    ↓
notifier 飞书卡片告警坤哥
    ↓
坤哥审核决定是否放行
```

### 场景3：Web4爬虫访问受限URL
```
Web4 Controller.research("paypal.com")
    ↓
IronRuler.is_access_allowed() 检查
    ↓
返回 False（PayPal在黑名单）
    ↓
请求被拒绝
```

## 比赛话术模板

### 架构介绍
> "我们的系统借鉴了 MemGPT 的虚拟内存分页、GraphRAG 的知识图谱、Claude Code 的三层记忆架构，形成了自己的 AI Agent 记忆与安全体系。"

### 安全体系
> "安全是三层防护——网络层（IronRuler）、执行层（Web4沙箱）、文件层（Clawkeeper）。即使AI试图绕过一层，第二层、第三层还能兜底。"

### 反黑箱机制
> "所有AI执行的任务都会实时推送飞书卡片给用户，确保AI的每一步都可观测、不黑箱。"
